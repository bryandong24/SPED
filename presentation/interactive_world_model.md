# Talking to a World Model

### Real-time, audio-steered video generation — **no extra training**

Built on gino's interactive world model (LongLive / Wan2.1-1.3B)

## The world model we started from

- **LongLive-1.3B** — autoregressive *streaming* video model (DMD base + LoRA on **Wan2.1-T2V-1.3B**).
- Generates video **chunk-by-chunk forever**: 3 latent frames/chunk (~0.75 s), 4 denoise steps, KV-cache over past frames (`local_attn_size=12, sink_size=3`).
- **Real-time:** ~12–18 FPS on a single H100 at 480×832 (16 FPS content).
- Already interactive via **text prompts** swapped mid-rollout.

> The question: can we drive it with **audio/voice** instead of typing — live, and without retraining the video model?

<!--

## The key insight: the text interface is a free hot-swap point

The model conditions on text via a **frozen umT5 encoder**:

`prompt → [≤512 tokens, 4096] sequence → cross-attention in every DiT block`

- Encoded **once per prompt**, then reused as a **frozen constant** across all denoise steps.
- Cached behind an `is_init` flag → flip it + pass new embeddings = **clean swap**, *zero per-step cost*, past-frame KV cache untouched.

**So: anything that can produce text can steer the world — at no video-side cost.**

<!--

## The main point: **the KV cache _is_ the world's state**

Two caches, two jobs — steering edits one and leaves the other running.

---

## Two caches, two jobs

**① Self-attention KV cache — the world's *visual memory*.**
Keys/values of the last **12 latent frames** (~3 s), + a pinned **3-frame "sink"** anchor. Every new frame is generated by **attending into it** → geometry, subject, lighting persist.

**② Cross-attention text cache — the *control input*.**
The umT5 prompt, behind an `is_init` flag. **Swap = flag flip + re-encode, zero per-step cost — cache ① untouched.**

> Audio swaps **②**. The world keeps flowing out of **①**.

<!--
Speaker: The state of the world is not in the weights and not in the prompt — it's
in cache ①, the rolling frame memory. Steering only touches cache ②.
-->

---

## Why steering = **morph, not cut**

On a prompt swap, the next chunk attends to:

- **old-world geometry + subject** (self-attn KV ①) — *under* —
- **new-world instruction** (swapped cross-attn ②)

The model **reconciles** them → bends the existing scene toward the new words.
The dog is *in the frame cache*, so it **persists**; the meadow morphs to snow around it.

**Recache-on-switch** (`_recache_after_switch`): the surviving window frames are re-run under the new prompt at ~zero noise → their K/V are refreshed "through" the new prompt → a **continuous join**, not a discontinuity.

**Momentum is a knob:** window size = how long the old world lingers (~3 s to fully establish a new scene); the 3 sink frames keep global identity stable. Smaller window → snappier swaps; larger → smoother, slower.

## Our contribution: training-free audio steering

**Cascade:** 🎙️ → 📝 → 🔤 → 🌍

```
mic / narration  →  streaming ASR  →  text prompt  →  umT5 (existing)  →  hot-swap context per chunk
```

- **Zero video-side training.** Reuses the model's own text interface.
- ASR (faster-whisper today) → utterances grouped by silence → per-chunk active prompt.
- Audio time ↔ video time 1:1; the scene becomes whatever is being **spoken**.

*Why not audio→embedding directly?* No prior art maps audio to a umT5 **sequence** (existing adapters target CLIP pooled space). A projector would be a multi-week, multi-GPU-day training build — **the cascade gets it for free, today.**

<!--

the text encoder that's already there. Training-free, and it works now.
-->

## It works: Phase 1 — dynamic injection

- Prompt switch at **arbitrary chunks** (decoupled from the repo's fixed switch points).
- **18.2 FPS** — faster than 16 FPS real-time.
- Re-encode + recache on switch: **~0.5 s**.

## Phase 2 — audio → video, end to end

- Narration: *"sunny meadow → snowy night → neon city"*
- Video **steered through all three scenes in sync** with the speech.
- Switches at 12.0 s and 23.2 s, **~0.5 s lag** after the spoken command.
- **12.9 FPS** with ASR in the loop. → `demo_audio_driven.mp4`

## The smooth emergent properties

- **Soft-ramp transitions, for free.** A *growing* live transcript is a stream of small prompt deltas → a natural gradual ramp, gentler than a hard text cut. Transitions **morph rather than snap**.
- **Subject persistence across scene changes.** The dog stays the same dog through meadow → snow → neon — the self-attention KV cache carries "world momentum" the new prompt bends rather than erases.
- **Coherent same-scene action switching** (e.g. action changes on a held subject) stay smooth.

> These weren't engineered in — they **emerge** from swapping a frozen text context over a model with persistent frame memory.

## Why this is the right design

| | Cascade (ours) | Audio→embedding projector |
|---|---|---|
| Video training | **none** | retrain/adapt |
| Build time | **working now** | weeks, multi-GPU-day |
| Latency to scene change | **~0.5 s** | unknown |
| Fits umT5 [L,4096] interface | **native** | needs new projector |
| Non-verbal acoustics (timbre/ambience) | text-limited | possible (future) |

Training-free **wins on everything except non-verbal acoustic control** — which is the one place a learned projector would later add value.
